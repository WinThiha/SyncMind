API Authentication Failure on Mobile App Login

In Progress
keyboard_double_arrow_up High Priority
person Assigned to Sarah J.

The issue stems from the recent OAuth2 token expiration logic update. Mobile clients are caching tokens longer than the server expects. Sarah J. identified the discrepancy in auth_middleware.go.

Suggested Fix: Implement a refresh token check pre-flight on the mobile client before initiating the primary API call to the protected endpoints.

Users are reporting that they are unable to log in to the mobile application. The error message displayed is "Authentication Failed. Please try again."

Steps to reproduce:

  1. Open the mobile app on iOS or Android.
  2. Enter valid credentials (e.g., testuser / password123).
  3. Tap 'Log In'.

Expected result: User is redirected to the dashboard.

Actual result: A toast message appears with the authentication error, and the user remains on the login screen.

// Log snippet from server
[ERROR] 2023-10-27T10:00:00Z auth_handler.go:45: Token validation failed: token expired
